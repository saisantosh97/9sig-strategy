# 9sig Strategy — Project Context for Claude

## Project Overview

**9sig Strategy** is a Python-based, rules-driven quarterly investing system designed to automate portfolio management for **TQQQ** (a 3× leveraged NASDAQ-100 ETF). It includes a backtesting engine, an interactive quarter-end decision calculator, and an automated HTML report generator.

## Strategy Rules

The 9sig strategy is a **rules-based quarterly rebalancing** approach:

### 1. Base Rule — 9% Quarterly Growth Target
- At each quarter end, check if ETF grew ≥9% from the previous quarter-end value.
- **Above target:** Sell excess ETF, move proceeds to cash.
- **Below target:** Buy more ETF using available cash.
- **At target:** Hold — no action needed.

### 2. 30-Down Rule — Crash Protection
- Triggered when the current price is ≥30% below its 2-year high.
- **Effect:** Suppresses sell signals even if the growth target is exceeded.
- Still allows buying if below target.
- After **2 consecutive skipped sell signals**, resets to 60/40 ETF/Cash allocation.

### 3. 100-Up Rule — Profit Lock
- Triggered when a single quarter's gain is ≥100%.
- **Effect:** Immediately rebalances the portfolio to the original 60/40 allocation.
- Purpose: Lock in extreme gains before a potential reversal.

## Key Files & What They Do

### `simulate.py`
Runs a full historical backtest using TQQQ.csv. Outputs:
- `results.csv` — Quarter-by-quarter portfolio state and decisions
- `results.png` — 3-panel chart: equity curve, ETF/cash allocation, drawdown

### `calculator.py`
Interactive CLI tool for real-world use at each quarter end. Accepts:
- ETF start/end values, cash balance, 30-Down status
- Price inputs (current price, previous price, 2-year high) or just growth % + flag

Outputs an action recommendation (Buy/Sell/Hold) with trade amounts and updated portfolio state.

### `generate_report.py`
Reads `results.csv` and builds a polished, beginner-friendly `index.html` report with:
- Embedded base64 charts
- Strategy explanation in plain language
- Key metrics (CAGR, Sharpe ratio, max drawdown)
- Quarter-by-quarter results table with color-coded action badges
- Risk warnings and disclaimers

### `config.yaml`
Central configuration for all strategy parameters. Modify here to test variations without touching source code.

## How to Run

```bash
# 1. Run the backtest
python TQQQ/simulate.py

# 2. Generate the HTML report
python TQQQ/generate_report.py

# 3. Use the quarter-end calculator interactively
python TQQQ/calculator.py

# 4. Or use it in CLI/batch mode
python TQQQ/calculator.py --etf-start 10000 --etf-end 11200 --cash 4000
```

## Notes for Future Claude Sessions

- All strategy logic lives in `simulate.py` and is reused by `calculator.py`.
- The `config.yaml` is the single source of truth for strategy parameters — prefer modifying it over hardcoding values.
- `index.html` is auto-generated — do not manually edit it.
- `results.csv` and `results.png` are output artifacts — safe to regenerate.
- TQQQ.csv is the raw data source — handle with care (do not overwrite historical data).
- The project is educational/backtesting in nature — it does not execute live trades.
